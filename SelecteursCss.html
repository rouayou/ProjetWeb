Selecteurs Css

Une des grandes richesses de CSS est la multiplicité des possibilités de sélection des éléments auxquels on veut attribuer un style donné.

Cette très grande diversité permet d’appliquer un style aussi facilement à tous les éléments, qu’à un unique élément isolé dans la page web, sans avoir à écrire la définition localement.

De plus, la combinaison de plusieurs sélecteurs dans la même déclaration permet de répondre à tous les besoins, même les plus complexes.

  1. Selecteurs à un seul élément

    Le sélecteur est constitué du nom de l’élément sans les caractères de début < et de fin de balise /<.

    Exemple :

    p {color : yellow;}

    Nous pouvons appliquer le même style à plusieurs éléments différents en les énumérant et en les séparant par une virgule dans le sélecteur.

    Exemple :

    h1,div,p {color : black ; background-color : red;}

  2. Le sélecteur universel

    Pour appliquer un style à tous les éléments, nous utiliserons le sélecteur universel *.

    Exemple :

    *{background-color: yellow;}

    On peut modifier cette propriété pour un élément particulier, en la redéfinissant uniquement pour celui-ci, par exemple :

    *{background-color: yellow;} p{background-color: gray;}

  3. Sélecteurs d‘identifiant (ID)

    Chaque élément peut avoir un attribut id qui doit être uniquedans une page donnée.

    Nous pouvons écrire un style qui ne sera applicable qu’à l’élément dont l’id a une valeur précise en donnant cette valeur au sélecteur (comme pour une classe) et en le faisant précéder du caractère (#).

    Exemple :

    #bleu {color: white; background-color: blue;}

  4. Sélecteur par contexte

    Permet d’appliquer un style en fonction de la position relative de l’élément par rapport à un autre dans la hiérarchie des éléments de la page. Ce type de sélecteur est dit contextuel.

    Syntaxe:

    element_parent element_enfant{ propriété1 : valeur1 ; propriété2 : valeur2 ;...; propriétéN : valeurN }

  5. Les sélecteurs parent-enfant directs

    Nous pourrons préciser le sélecteur précédent en n’appliquant qu’un style à un élément à condition qu’il soit un enfant direct d’un autre élément et non plus un descendant indirect.

    Syntaxe :

    element_parent > element_enfant{ propriété1 : valeur1 ; propriété2 : valeur2 ;...; propriétéN : valeurN }

  6. Les sélecteurs d’éléments adjacents ou frères

    Les éléments adjacents ou frères sont des éléments d'un seul père.

    • Pour sélectionner la relation entre deux éléments frères, nous disposons du sélecteur ~ :

      élément1~élément2 { propriété1 : valeur1 ; propriété2 : valeur2 ;...; propriétéN : valeurN }

    • Pour sélectionner la relation entre deux éléments frères consécutifs, nous disposons du sélecteur + :

      élément1+élément2 { propriété1 : valeur1 ; propriété2 : valeur2 ;...; propriétéN : valeurN }

  7. Le sélecteur :not

    Lesélecteur: not correspond à chaque élément qui n'est pas l'élément spécifié par cet sélecteur.

    Syntaxe :

    :not(élément){ propriété1 : valeur1 ; propriété2 : valeur2 ;...; propriétéN : valeurN }

  8. Sélecteur d’attribut

    On peut définir un sélecteur d’attribut sans en préciser la valeur.

    Exemple :

    *[title] {background–color: yellow;}

    Il est possible de sélectionner plusieurs attributs pour un élément en faisant suivre son nom de plusieurs attributs entre crochets.

    Exemple :

    h2[title][id]{background-color: yellow;}

  9. Sélecteur de valeur d’attribut

    C’est une méthode qui sélectionne les balises ayant un attribut avec une valeur donnée.

    Exemple :

    input[type="button"] { background-color: yellow; }

    Il est possible de sélectionner plusieurs attributs et leurs valeurs en utilisant la syntaxe :

    élément[attribut1="valeur1"][attribut2="valeur2"] {propriété1 : valeur1 ; propriété2 : valeur2 ;...; propriétéN : valeurN}
